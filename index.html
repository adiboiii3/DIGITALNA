Šta su Multivibratori?

Elektronski sklopovi s ukrštenom povratnom spregu između dva NPN tranzistora koji prelaze između zasićenja i zakočenja

// Osnova rada — prekidački režim

Svaki tranzistor radi kao elektronski prekidač — ili je potpuno uključen (zasićenje) ili potpuno isključen (zakočenje). Između ova dva stanja nema sredine — to je ključna osobina multivibratora.

// 3 vrste multivibratora
  • Astabil
    0 stabilnih stanja — sam beskrajno osciluje između dva kvazistabilna. Nema potrebe za okidačem. Primjena: generator takta, blinker, oscilator
  • Monostabil
    1 stabilno + 1 kvazistabilno — treba negativni impuls da pređe u kvazi. Sam se vraća u stabilno. Primjena: tajmer, impuls zadanog trajanja
  • Bistabil
    2 stabilna stanja — flip-flop. Treba 2 okidača. Ostaje u stanju dok ne dobije novi impuls. Primjena: memorijska ćelija, registar
// Dvije brzine promjene — ključno!
  • RC·C
    Brza promjena — kondenzator se puni kroz mali kolektorski otpornik RC (red kΩ) → strme ivice impulsa, skoro trenutna promjena
  • RB·C
    Spora promjena — kondenzator se prazni kroz veliki bazni otpornik RB (red 10·kΩ) → eksponencijalni rast, određuje trajanje kvazistabilnog stanja
  • 0.693
    = ln(2) — dolazi iz rješavanja eksponencijalne jednadžbe punjenja/pražnjenja kondenzatora. Otuda sve formule T = 0.693·R·C

// Zašto kondenzator = pozitivna sprega?
TR1 počne voditi više
→ Kolektor TR1 pada
→ Kondenzator prenosi pad na bazu TR2 (brza promjena!)
→ TR2 se isključuje
→ Kolektor TR2 raste
→ Kondenzator prenosi rast na bazu TR1
→ TR1 se još više uključuje → LAVINA! → zasićenje

Kondenzator = pozitivna sprega jer pojačava promjenu umjesto da je suzbija.

// Asinhroni vs. sinhroni ulazi flip-flopa
  • Sinhroni
    D, J, K, S, R — djeluju SAMO kad je sinhronizovano s taktom CP. Van takta ignorišu signal.
  • Asinhroni
    SET i RESET (Preset, Clear, MR) — djeluju ODMAH bez obzira na takt. CLK ih ne zanima.

Astabilni Multivibrator

Nema stabilnih stanja — sam osciluje beskrajno. Generator takta, blinker. Samo pozitivna povratna sprega u oba smjera.

// Shema — ukrštena pozitivna sprega
// Formule trajanja i frekvencije
T₁ ≈ 0.693 · RB2 · C2
T₂ ≈ 0.693 · RB1 · C1
T = T₁ + T₂
f = 1/T = 1/(T₁ + T₂)
Zašto 0.693? Rješavanjem eksponencijalne jednadžbe pražnjenja kondenzatora dobijemo da je T = τ·ln(2) = τ·0.693. Ovdje je τ = RB·C.

Simetrični signal (duty cycle 50%): ako je RB1·C1 = RB2·C2, onda T1 = T2 i signal je savršeno simetričan.
// Komponente i uloge
  • RC1,RC2
    Kolektorski otpornici (red kΩ) — ograničavaju struju, mala tau → brzo punjenje C → strme ivice impulsa
  • RB1,RB2
    Bazni otpornici (red 10·kΩ) → spora promjena → određuju trajanje kvazistabilnih stanja!
  • C1,C2
    Kondenzatori — propuštaju brze promjene, realizuju pozitivnu spregu, srce osciliranja
  • TR1,TR2
    NPN bipolni tranzistori — rade kao elektronski prekidači (zasićenje ili zakočenje)

Monostabilni Multivibrator

Jedno stabilno + jedno kvazistabilno stanje. Treba negativni okidač. Sama se vraća u stabilno stanje nakon T = 0.693·RB·CB.

// Shema — pozitivna + negativna sprega
// Ključne razlike vs. Astabilni
  • Astabil
    Samo pozitivna sprega u OBA smjera (C1 i C2) → nema stabilnog stanja, sam osciluje
  • Monostabil
    Pozitivna sprega via CB (Kol.TR2→Baza TR1) + Negativna via R1-R2 (Kol.TR1→Baza TR2) → jedno stabilno!
  • RB→UCC
    Baza TR1 direktno vezana na UCC → odmah po priključenju napajanja TR1 u zasićenje (stabilno stanje)
  • UCE1
    Skoro trenutan skok (bez kondenzatora u kolektorskom kolu TR1 → nema RC konstante)
  • UBE1
    Eksponencijalni rast s τ = RB·CB → polako raste dok ne dostigne Uγ → povratak u stabilno
  • UBE2
    NEMA eksponencijale! Samo otpornici R1-R2 u kolu → nema RC → skoro trenutna promjena
T = 0.693 · RB · CB

// Tok analize monostabilnog MV
1
Stabilno stanje — TR1 zasičen (RB→UCC), TR2 zakočen (R1-R2 dijeli napon). CB napunjen na UCC−UBEzas. Sklop miruje.
2

3

4

5
Povratak u stabilno — kad UBE1 dostigne Uγ, TR1 ponovo počne voditi → lavinski prelaz nazad u stabilno stanje.
// Riješen zadatak
Analizirati monostabil: UCC=6V, RB=15kΩ, CB=1μF, RC=1kΩ, R1=10kΩ, R2=80kΩ, hFE=30. Odrediti trajanje kvazistabilnog stanja.
// Rješenje:
Stabilno stanje:
ICzas2 = (UCC − UCEzas)/RC = (6−0.3)/1kΩ = 5.7 mA
IBzas2 = (UCC − UBEzas)/RB = (6−0.7)/15kΩ = 0.353 mA
Provjera zasićenja: IBzas2 = 0.353 ≥ ICzas2/hFE = 5.7/30 = 0.19 mA ✓

UBEzak1 = UCEzas·R2/(R1+R2) − UBB·R1/(R1+R2) = 0.3·80/90 − 6·10/90 = 0.267 − 0.667 = −0.4V (dobro zakočen!)

Trajanje kvazi stanja:
T = τ·ln2 = 0.693·RB·CB = 0.693 · 15·10³ · 10⁻⁶ = T ≈ 10.4 ms

Bistabilni Multivibrator

Dva stabilna stanja. Flip-flop. Treba dva okidača. Ostaje u stanju dok ne dobije impuls za promjenu. Osnova svih memorija.

// Karakteristike bistabila
  • 2 stabilna
    Za razliku od astabilnog (0) i monostabilnog (1), bistabil ima 2 stabilna stanja i u njima ostaje beskonačno
  • Okidač
    Treba okidač (impuls) za prelaz iz jednog u drugo stanje. Bez okidača ostaje u tekućem stanju.
  • Primjena
    Flip-flop, memorijska ćelija, brojač, registar, detektor ivice signala
  • Negativna
    Ima negativnu spregu u OBA smjera → oba tranzistora stabilno drže stanje
UCEzak = UCC − (UCC − UBEzas)/(RC + R1) · RC
UBEzak = UCEzas · R2/(R1+R2) − UBB · R1/(R1+R2)
// Riješen zadatak
Analizirati bistabil: UCC=10V, −UBB=5V, RC=1.5kΩ, R1=6.8kΩ, R2=47kΩ, RB=15kΩ, hFE=50.
// Rješenje (pretpostavimo TR1 u zasićenju):
ICzas ≈ (UCC − UCEzas)/RC = (10−0.3)/1.5 = 6.47 mA
IBzas = (UCC−UBEzas)/(RC+R1) − (UBEzas+UBB)/R2 = (10−0.8)/8.3 − (0.8+5)/47 = 1.11 − 0.124 = 0.986 mA
hFE ≫ ICzas/IBzas = 6.47/0.986 → TR1 duboko u zasićenju ✓

UCEzak (TR2 zakočen): = 10 − (10−0.8)/(1.5+6.8)·1.5 = 10 − 1.66 = 8.34V
UBEzak = 0.3·47/53.8 − 5·6.8/53.8 = 0.262 − 0.632 = −0.37V (dobro zakočen ✓)

Usporedba Multivibratora

Sve što trebaš znati za ispit na jednom mjestu

// Za ispit — astabilni

Astabilni multivibrator nema stabilnih stanja. Sam osciluje. Povratna sprega je samo pozitivna (C1 i C2 u oba smjera). Tranzistori se naizmjenično zasičuju i zakočuju.

T₁ = 0.693 · RB2 · C2
T₂ = 0.693 · RB1 · C1
T = T₁ + T₂    f = 1/T
  • UCE
    Eksponencijalni porast s τ = RC·C (puni se brzo → strme ivice)
  • UBE
    Eksponencijalni rast (za oba!) s τ = RB·C → određuje trajanje stanja
  • Init
    Nestabilno → sam ide u kvazi → osciluje beskonačno
  • 0.693
    = ln(2) iz eksponencijalne matematike pražnjenja C
// Za ispit — monostabilni

Monostabilni ima jedno stabilno (RB→UCC drži TR1 u zasićenju) i jedno kvazistabilno. Treba negativni okidač. Sama se vraća u stabilno.

T = 0.693 · RB · CB
  • CB
    Pozitivna sprega → kvazistabilno (Kol.TR2→Baza TR1)
  • R1-R2
    Negativna sprega → stabilno (Kol.TR1→Baza TR2)
  • UCE1
    Skoro trenutan skok — nema C u kolekt. kolu TR1!
  • UBE1
    Eksponencijalni rast, τ = RB·CB
  • UBE2
    NEMA eksponencijale! Samo otpornici R1-R2

Šta su Registri?

Memorijski sklopovi građeni od D flip-flopova. Mogu zapamtiti n bita (n = broj FF-ova).

// Zašto D flip-flop za registre?
Registri se grade od D flip-flopova jer D ff ima jednu ključnu osobinu: signal sa ulaza D prenosi se direktno na izlaz Q bez ikakve promjene (Q = D). To je najvažnije za memoriju — podatak koji se upiše mora biti ispisan tačno onakav kakav je bio. Kod JK ili SR ff-a postoji transformacija ulaza, pa se ne mogu direktno koristiti za memoriju.
  • Sinhroni ulazi
    D, J, K, S, R — djeluju SAMO uz takt CP. Promjena se dešava na uzlaznoj (↑) ili silaznoj (↓) ivici CLK-a.
  • Asinhroni ulazi
    SET i RESET (Preset, Clear, MR) — djeluju ODMAH bez obzira na takt. Koristimo ih za inicijalizaciju registra.
// Podjela registara po načinu upisa/ispisa
  • P upis, P ispis
    STACIONARNI REGISTAR — neovisni FF-ovi, svaki ima vlastiti ulaz i izlaz, nema fizičke veze između FF-ova
  • S upis, S ispis
    POMJERAČKI (Shift) — serijski ulaz i izlaz, izlaz prethodnog je ulaz sljedećeg
  • P upis, S ispis
    Paralelno se upiše, serijski izlazi
  • S upis, P ispis
    Serijski ulaz, paralelni izlaz
// Asinhroni vs Sinhroni ulazi — na primjeru

Ako su svi FF-ovi bili u stanju logičke 1 i dovedemo MR (Master Reset) signal — logičku 0 na asinhroni RESET ulaz:

Asinhroni RESET djeluje odmah, bez obzira na CLK.
Svi FF-ovi prelaze iz 1 → 0 momentalno.
Tek nakon toga rješavamo vremenski dijagram.

Ovo je ključno u zadacima — uvijek prvo primijeniti asinhroni ulaz, pa tek onda pratiti CLK ivice!

// Referentne tačke u vremenskom dijagramu

Referentne tačke su uzlazne ivice (↑) CLK-a (za FF-ove s uzlaznom ivicom). To su jedini trenuci kada FF može promijeniti stanje. Između dvije referentne tačke, FF ignoriše sve promjene na ulazu D.

Važno! Ako se signal na ulazu D promijeni između dvije uzlazne ivice CLK-a, FF tu promjenu ne vidi. FF "gleda" D samo u trenutku uzlazne ivice!

Stacionarni Registar

Paralelan upis, paralelan ispis. Svaki FF je nezavistan. Nema fizičke veze između FF-ova. Rješavamo svaki FF zasebno.

// Shema — 3-bitni stacionarni registar
// Pravila rješavanja
1
Označi sve uzlazne ivice CLK-a — to su referentne tačke
2
Za svaki FF zasebno: na svakoj uzlaznoj ivici pročitaj vrijednost D ulaza
3
Q postaje ta vrijednost i ostaje do sljedeće uzlazne ivice
4
Promjene D između dvije ivice se ignorišu!
Ključna razlika od pomjeračkog: Stacionarni se rješava FF po FF (nezavisno). Pomjerački se rješava ivica po ivica (zavisno — Q prethodnog je D sljedećeg).
// Interaktivni simulator — korak po korak
Pritisni Sljedeća ivica da vidiš šta se dešava na svakom taktu. Svaki FF se rješava zasebno!
// Riješen primjer — stacionarni registar
Svi FF-ovi su bili u stanju 1 (setovani). MR djeluje → resetuje sve na 0. Na prvoj uzlaznoj ivici D1=1, D2=0, D3=1. Na drugoj D1=0, D2=1, D3=0.
// Rješenje:
MR=0 (asinhroni) → odmah svi Q=0 (bez obzira na CLK)

1. uzlazna ivica CLK: Q1←D1=1, Q2←D2=0, Q3←D3=1
2. uzlazna ivica CLK: Q1←D1=0, Q2←D2=1, Q3←D3=0

Ako se D promijeni između dvije ivice — FF to ne vidi!
Rješavamo svaki FF zasebno i nezavisno.

Pomjerački (Shift) Registar

Serijski upis, serijski ispis. Izlaz prethodnog FF-a = ulaz sljedećeg. Podatak "putuje" kroz registar 1 korak po taktu.

// Shema — 4-bitni pomjerački registar
// Zašto koristimo PRETHODNO stanje?
Ključno pravilo pomjeračkog registra:

U trenutku uzlazne ivice CLK-a, signal sa izlaza prethodnog FF-a još nije stigao da se prenese na novi izlaz (postoji kašnjenje propagacije). Zato svaki FF "vidi" stanje koje je prethodni FF IMAO PRIJE te ivice.

Zato rješavamo ivica po ivica, a ne FF po FF!
// Simulator — unesi bit na serijski ulaz
Pritisni Unesi 1/0 da ubacuješ bit na serijski ulaz. Svaki novi takt, bit se pomjera desno.
// Vremenski dijagram pomjeračkog registra
Podatak unijet na SI pojavljuje se na SO (serijski izlaz) nakon n taktova (n = broj FF-ova = 4). Registar uvodi kašnjenje od n taktova.
// Riješen primjer — pomjerački registar ivica po ivica
Trobitni pomjerački registar, svi FF-ovi resetovani (0,0,0). Na serijskom ulazu SI slijed: 1,0,1,1,0,0...
// Rješenje (Q1=SI ulaz, Q2=prethodni Q1, Q3=prethodni Q2):
Početak: Q1=0, Q2=0, Q3=0
1. ivica (SI=1): Q1←SI=1, Q2←prev.Q1=0, Q3←prev.Q2=0 → 1,0,0
2. ivica (SI=0): Q1←0, Q2←prev.Q1=1, Q3←prev.Q2=0 → 0,1,0
3. ivica (SI=1): Q1←1, Q2←0, Q3←1 → 1,0,1
4. ivica (SI=1): Q1←1, Q2←1, Q3←0 → 1,1,0

Vidi se da 1 unijet na 1. taktu se pojavljuje na SO (Q3) na 3. taktu!

Kružni Registar — Prstenasti i Johnson

Posebna vrsta pomjeračkog registra gdje je izlaz posljednjeg FF-a vraćen na ulaz prvog. Registar koji broji!

// Prstenasti (Ring) brojač

Izlaz Q posljednjeg FF-a direktno se vraća na ulaz prvog. Sa n FF-ova — n stanja, dijeli frekvenciju sa n. Broji od 0 do n−1.

VAŽNO! Prstenasti brojač mora imati barem JEDNU jedinicu upisanu u registar! Ako su svi 0, kružit će samo nule i neće biti vidljivo brojanje. Počinjemo paralelnim upisom jedinice u prvi FF.
Broji: 0 do n-1    Dijeli f sa: n
Sekvenca (1000): 1000→0100→0010→0001→1000...
// Johnsonov (Twisted Ring) brojač

Jedina razlika: vraća se Q̄ (komplement) posljednjeg FF-a. Sa n FF-ova — 2n stanja, dijeli frekvenciju sa 2n. Broji od 0 do 2n−1.

Zašto 2n stanja?
Komplement "hvata" i fazu punjenja (0→1) i fazu pražnjenja (1→0), čime se udvostručava broj stanja u odnosu na prstenasti.
Broji: 0 do 2n-1    Dijeli f sa: 2n
Sekvenca (4 FF): 0000→1000→1100→1110→1111→0111→0011→0001→0000

// Rješenje (Q3 posljednji, Qd→Q0):
Početno: Q3=1, Q2=1, Q1=0, Q0=1
Takt 1: QD(=1)→Q0, Q0(=1)→Q1, Q1(=0)→Q2, Q2(=1)→Q3: 1,1,1,0 (čitaj Q3Q2Q1Q0)

Šta su Brojači?

Sekvencijalni sklopovi koji broje impulse CLK-a. Sa istim brojem FF-ova postižu znatno veće dijeljenje frekvencije nego registri.

// Zašto JK/T flip-flop za brojače?
Kod registara koristimo D ff (Q=D). Kod brojača koristimo JK flip-flop (ili T koji je JK s kratko spojenim ulazima J=K=T). Razlog: ako su J=K=1, JK ff invertuje prethodno stanje na svakom taktu. To nam treba za brojanje!
  • JK, J=K=1
    Svaki put kad dobije takt impuls → invertuje prethodno stanje. Q0: 0→1→0→1... = savršeno za bit najmanje vrijednosti!
  • T ff, T=1
    Ista osobina — T=1 invertuje, T=0 zadržava. Koristi se za sinhroni i sekvencijalni brojač.
// Podjela brojača
  • Asinhroni
    Serijski/ripple — izlaz prethodnog FF→CLK sljedećeg. Promjene se dešavaju "u kaskadi". Sporiji ali jednostavniji.
  • Sinhroni
    Paralelni — CLK vezan na sve FF-ove istovremeno. AND kola određuju kada koji FF mijenja stanje. Brži, precizniji.
  • Naprijed
    Broji 0,1,2,3... od manjeg prema većem
  • Nazad
    Broji ...3,2,1,0 od većeg prema manjem
  • Up-Down
    Može oboje zavisno od DIR signala

// Shema — 4-bitni asinhroni brojač naprijed
// Pravilo rada
1

2

3

Asinhroni Binarni Brojač Nazad

Jedina razlika od naprijed: signal sa Q̄ (komplement) prethodnog ide na CLK sljedećeg. Broji 15→14→...→0→15.

// Zašto Q̄ obrće smjer?
Brojač naprijed: signal sa Q→CLK
Kada Q0: 0→1 (pozitivna), Q̄0: 1→0 (negativna)
Kada Q0: 1→0 (negativna), Q̄0: 0→1 (pozitivna)

Kod naprijed, FF1 se okida na negativnu Q0 (1→0)
Brojač nazad: signal sa Q̄→CLK
Kada Q0: 0→1, Q̄0: 1→0 → okida FF1!
Kada Q0: 1→0, Q̄0: 0→1 → ne okida FF1

Rezultat: FF1 se okida na suprotnom momentu → broji NAZAD!
Važno iz predavanja: I dalje čitamo izlaze sa GLAVNIH Q izlaza (Q0, Q1, Q2, Q3), ne sa Q̄! Samo signal za CLK sljedećeg uzimamo sa Q̄.
// Šta se dešava iz 0000?

Kada su svi u 0 i dođe silazna ivica CLK-a:

FF0: 0→1 (pozitivna na Q). Znači Q̄0: 1→0 (negativna!). FF1 se okida → 0→1. Q̄1: 1→0 → FF2 se okida → 0→1. Q̄2: 1→0 → FF3 se okida → 0→1.

Iz stanja 0000 prelazimo direktno u 1111 (=15)! Onda broji 15,14,13...
// Primjer prelaza 8→7
U stanju 1000 (=8), dođe silazna ivica CLK-a. Šta se dešava?
// Rješenje:
Stanje: Q3=1,Q2=0,Q1=0,Q0=0
Silazna ivica → Q0: 0→1 (+), Q̄0: 1→0 (−) → FF1 se okida
FF1: Q1: 0→1 (+), Q̄1: 1→0 (−) → FF2 se okida
FF2: Q2: 0→1 (+), Q̄2: 1→0 (−) → FF3 se okida
FF3: Q3: 1→0 (−), Q̄3: 0→1 (+) → FF4 NE okida

Sinhroni (Paralelni) Brojač

CLK na svaki FF istovremeno. AND kola određuju kada koji FF može promijeniti stanje. Brži i precizniji od asinhronog.

// Kako radi sinhroni brojač?

Svi FF-ovi reaguju na isti CLK istovremeno. Ali svaki FF ima JTK ulaz koji može biti 0 ili 1:

  • JK=0
    FF zadržava prethodno stanje
  • JK=1
    FF invertuje prethodno stanje
Kada se koji FF mijenja?

FF0: uvijek (JK0=1 stalno)
FF1: samo ako je Q0=1 (AND: Q0→JK1)
FF2: samo ako su Q0=1 I Q1=1 (AND: Q0·Q1→JK2)
FF3: samo ako su Q0=1 I Q1=1 I Q2=1 (AND: Q0·Q1·Q2→JK3)
// Primjer — prelaz 7→8 u sinhronom brojaču
Sinhroni 4-bitni brojač u stanju 0111 (=7). Dođe negativna ivica CLK-a. Šta se dešava?
// Rješenje:
Stanje: Q3=0,Q2=1,Q1=1,Q0=1

JK0=1 → FF0 mijenja: 1→0
JK1=Q0=1 → FF1 mijenja: 1→0
JK2=Q0·Q1=1·1=1 → FF2 mijenja: 1→0
JK3=Q0·Q1·Q2=1·1·1=1 → FF3 mijenja: 0→1

Novo: Q3=1,Q2=0,Q1=0,Q0=0 = 1000 (=8) ✓
Svi mijenjaju u ISTOM taktu → nema kaskadnog kašnjenja!
// Asinhroni vs Sinhroni — vizuelno
Sinhroni prednost: Kod asinhronog, promjena od FF0 do FF3 traje 4×tpd (4 puta kašnjenje propagacije). Kod sinhronog, svi FF-ovi mijenjaju odjednom → samo 1×tpd + kašnjenje AND kola. Znatno brže!
// Sinhroni simulator
Sinhroni brojač — svi FF-ovi reaguju istovremeno na isti CLK!
